# Payment Deployment & Configuration Checklist

This document collects steps and environment variables to deploy and verify the Konnect payment integration.

## Required environment variables
- `PAYMENT_GATEWAY` (default: `konnect`)
- KONNECT_*:
  - `KONNECT_API_KEY`
  - `KONNECT_API_SECRET`
  - `KONNECT_SANDBOX` (true/false)
  - `KONNECT_WEBHOOK_SECRET`
- `PAYMENT_WEBHOOK_DISPATCH_SYNC` (set `true` for local test runs, `false` for staging/production)
- `PAYMENT_INITIATE_PER_MINUTE` (integer; controls rate limiter)

## Deployment checklist (Sandbox → Staging → Production)
1. Configure sandbox keys in environment and deploy to a staging branch.
2. Ensure `PAYMENT_WEBHOOK_DISPATCH_SYNC=false` and that a queue worker is running (e.g. `php artisan queue:work --tries=3`).
3. Start a worker and ensure failed job handling is configured (monitor `failed_jobs` table or external DLQ).
4. Register the Konnect webhook URL in the sandbox console and verify the callback payload against the configured webhook secret.
5. Run smoke tests: initiate a payment, complete checkout in sandbox, verify webhook is received and `payments` row is updated to `paid` and reservation/subscription reconciled.
6. Verify rate limiting by simulating repeated `POST /v1/payments/initiate` calls; tune `PAYMENT_INITIATE_PER_MINUTE` as required.
7. Enable monitoring and alerts: track webhook 5xxs, failed job counts, queue backlog, and payment reconciliation failures.
8. Flip to production keys and repeat validation steps.

## Observability & operational notes
- Webhooks should be processed quickly — the controller verifies signature and enqueues `ReconcilePaymentJob`.
- Workers should be configured with sensible retry/backoff and a dead-letter process for persistent failures.
- Add monitors on: failed_jobs, queue length, webhook 403/500 rates, and payment reconciliation failure logs.

## Quick commands
- Start queue worker:

```bash
php artisan queue:work --tries=5 --sleep=3 --timeout=90
```

- Run only payment tests locally (Pest/PHPUnit):

```bash
php artisan test --filter=Payment
```
